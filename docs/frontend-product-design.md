# QuantLoom·量梭 — 前端产品方案设计文档

> 股票行情分析与投资决策前端应用 · 完整产品方案

---

## 一、产品概述

### 1.1 产品定位

QuantLoom·量梭前端是一款**面向 A 股个人投资者的机构资金监控与辅助决策看板**。它不替代交易终端，而是作为"第二屏"——在传统的分时/K 线图之外，提供资金异动检测、AI 归因解读、信号置信度评估三层能力。

### 1.2 核心差异化

| 维度 | 传统行情软件 | QuantLoom |
|------|-------------|-----------|
| 数据视角 | 价格/成交量 | 主力资金流向 + 事件共振 |
| 信号来源 | 技术指标 | 规则引擎 + LLM 归因 |
| 可解释性 | 无/弱 | AI 生成一句话逻辑 + 证据链 |
| 操作闭环 | 看盘 → 人工决策 | 异动扫描 → 信号推送到人 → 反馈校准精度 |

### 1.3 用户画像

- **主要用户**：有 1-3 年 A 股交易经验的个人投资者
- **使用场景**：盘中每 10 分钟看一眼预警推送，午间/收盘后回顾 AI 分析日报，周末做持仓复盘
- **设备偏好**：盘中手机看推送，复盘用桌面浏览器

### 2.1 行情看板 (Dashboard)

**用途**：一屏总览今日市场全貌，快速感知"有没有值得关注的事"。

| 子模块 | 内容 |
|--------|------|
| 市场概览卡片 | 今日告警总数、P1 高风险数、AI 分析覆盖率、近 7 天日均 — 4 个统计数字卡片 |
| 异动趋势图 | ECharts 折线图：近 7 天每日告警量 × 按类型分色（breakout/accumulation/tail/event/sector） |
| 最新信号列表 | 最近 10 条告警的快速列表，点击进入详情 |
| 板块热度 | 热力图：行业板块 × 信号密度（替代方案：旭日图或矩形树图） |
| 资金总览 | 今日主力资金净流入 Top10 / 净流出 Top10（双栏对比条形图） |

### 2.2 信号中心 (SignalCenter)

**用途**：全量告警的查询、筛选、排序、批量操作。

| 子模块 | 内容 |
|--------|------|
| 筛选栏 | alert_type（放量上攻/底部吸筹/尾盘抢筹/事件驱动/板块联动）、risk_level（P1/P2/P3）、日期范围、股票代码搜索 |
| 信号列表 | 分页表格/卡片视图切换 — 每条含 code+name、触发时间、信号类型、置信度圆环、触发原因摘要 |
| 信号详情 | 点击进入 — 完整 AI 分析（summary、evidence、risk_points、action）、关联事件时间线、资金流快照、通知日志 |
| 批量操作 | 标记已读、导出 CSV、加入观察池 |
| 人工反馈 | 正确/错误/模糊 评审按钮 → 写入 `sq_alert_feedback` 闭环校准精度 |

### 2.3 持仓管理 (Portfolio)

**用途**：自选股/持仓股的异动监控与模拟收益跟踪。

| 子模块 | 内容 |
|--------|------|
| 自选股列表 | 手动添加/导入股票代码，显示最新价、涨跌幅、今日是否有异动信号 |
| 持仓录入 | 录入成本价、持仓数量 → 计算浮动盈亏 |
| 异动关联 | 自选股若触发信号，高亮标记并显示 AI 摘要 |
| 模拟收益对比 | 选取某只股票的某次告警时间点，查看 T+1/3/5 模拟收益 vs 大盘基准 |

### 2.4 消息预警 (NotificationCenter)

**用途**：管理推送规则和查看历史通知。

| 子模块 | 内容 |
|--------|------|
| 推送规则配置 | P1 即时推送（webhook/邮件）、P2 进入观察池、P3 仅入库 |
| 通知历史 | 按时间倒序展示所有已发送通知，含渠道、状态、内容摘要 |
| 推送渠道管理 | 企业微信/飞书/钉钉/邮件 的开关与配置状态 |

### 2.5 AI 分析共享组件

**用途**：被多个页面复用的核心展示组件，统一 AI 分析输出到可视化表达的转化逻辑。

### 3.1 色彩体系

```
┌─────────────────────────────────────────────┐
│  QuantLoom 金融风配色方案                      │
├──────────┬──────────┬────────────────────────┤
│ 角色      │ 色值      │ 用途                   │

│ 跌/卖出   │ #16A34A  │ 跌幅、卖出提示           │
│ 中性/持有  │ #6C757D  │ P3 标签、次要文字        │
│ 强调蓝    │ #2563EB  │ 主按钮、链接、选中态      │
│ 警示橙    │ #F57C00  │ P2 标签、中等风险         │
│ 边框      │ #E9ECEF  │ 分割线、输入框边框        │
│ 文字主色   │ #1A1A2E  │ 标题、正文              │
│ 文字次色   │ #6C757D  │ 辅助说明、时间戳         │
└──────────┴──────────┴────────────────────────┘
```

**关键规则**：
- 涨红跌绿（A 股市场惯例），不使用国际市场的"绿涨红跌"
- 置信度高于 0.7 用蓝色高亮，低于 0.4 用灰色弱化
- P1 红色标签 + 左侧红色竖线，P2 橙色标签，P3 灰色标签无特殊标记

### 3.2 图表方案

| 数据场景 | 图表选型 | 理由 |
|----------|---------|------|
| 告警趋势 (时间序列) | **ECharts 折线图** + 面积渐变 | 按类型分 5 条线，底部面积半透明，便于对比各类型占比变化 |
| 主力资金流向 | **水平对比条形图** | 左侧净流出 Top10（绿色），右侧净流入 Top10（红色），一目了然 |
| 板块热度 | **矩形树图 (Treemap)** | 板块面积 ∝ 信号密度，颜色深浅 ∝ 平均置信度 |
| 单股资金流 | **堆叠柱状图** | 超大单/大单/中单/小单四段堆叠，正负值分色 |
| 信号置信度 | **仪表盘/圆环 (Gauge)** | 0-100% 半圆，低于阈值红色弧线，高于阈值蓝色 |
| 关联事件 | **垂直时间线** | 沿时间轴排列新闻/公告/研报，左侧图标区分类型 |
| K 线图 | **TradingView 轻量图表** 或 ECharts candlestick | 仅在详情页作为辅助参考，不替代主交易终端 |

### 3.3 买卖信号可视化

信号从"数据"到"决策建议"的转化链路：

```
规则引擎告警 ──→ AI 归因分析 ──→ 置信度评分 ──→ 前端可视化

                                              ├── 推荐理由卡片 (AI summary + evidence 列表)
                                              ├── 操作建议标签 (watch / review / ignore)
                                              └── 模拟收益回测 (T+1/3/5 收益对比)
```

### 3.4 响应式策略

| 断点 | 宽度 | 布局变化 |
|------|------|---------|
| 手机 | < 768px | 单列布局；底部 Tab 导航（行情/信号/持仓/消息）；图表简化为迷你图；告警列表每屏 5 条 |
| 平板 | 768-1024px | 双列网格；侧边栏折叠为汉堡菜单；图表中等尺寸；分页 10 条/页 |
| 桌面 | > 1024px | 多列 + 侧边栏常驻；完整尺寸图表；分页 20 条/页；支持键盘快捷键 |

### 3.5 微交互

- **新信号到达**：顶部通知栏下滑 + P1 信号左侧闪烁红点（3 次后静止）
- **置信度悬停**：圆环 hover 显示历史精度、样本数、校准系数
- **列表加载**：骨架屏（灰色矩形占位）替代 spinner，减少布局跳动
- **反馈操作**：点击正确/错误按钮后，卡片滑出 + toast "评审已提交"

## 五、前端技术选型

### 5.1 推荐技术栈

| 层面 | 选型 | 版本 | 理由 |
|------|------|------|------|
| 框架 | **Vue 3** (Composition API) | ^3.5 | 用户指定；TS 支持好；Pinia 生态成熟 |
| 构建 | **Vite** | ^6 | 开发热更新 < 1s；生产 Rollup 打包 |
| 语言 | **TypeScript** | ^5.6 | 金融数据字段多，类型安全避免运行时错误 |
| 路由 | **Vue Router 4** | ^4.4 | hash history 模式（兼容无服务端配置的静态部署） |
| 状态 | **Pinia** | ^2.2 | Vue 3 官方推荐，模块化 store |
| HTTP | **Axios** | ^1.7 | 拦截器统一错误处理、请求去重、自动重试 |
| 图表 | **ECharts 5** | ^5.5 | 国内金融产品事实标准；K 线/热力图/桑基图内置支持；移动端手势 |
| CSS | **手写 CSS** + CSS 变量 | — | 白色金融风，不引入组件库避免样式冲突 |
| 图标 | **Lucide Vue** | ^0.4 | 轻量 SVG 图标库，tree-shaking |
| 日期 | **dayjs** | ^1.11 | 2KB 替代 moment.js |
| 部署 | FastAPI StaticFiles + SPA fallback | — | 单进程部署，无需 Nginx |

### 5.2 不引入的依赖及理由

| 排除 | 理由 |
|------|------|
| Element Plus / Ant Design Vue | 组件风格与金融白+高密度信息展示不兼容，引入后需要大量覆写样式 |
| Tailwind CSS | 新增构建依赖；金融场景定制化高（涨跌色、置信度条），手写更可控 |
| TradingView Charting Library | 需要授权；后端无历史 K 线数据源；K 线图非核心功能 |
| WebSocket | 用户当前数据源（AkShare/XTick）均为 HTTP 轮询；Celery 已处理定时扫描 |
